  <action>
Run the scaffold command in the project root:

```bash
npx create-next-app@16 . --typescript --tailwind --biome --app --src-dir --turbopack --import-alias "@/*"
```

This scaffolds into the current directory. If prompted about overwriting existing files (like .gitignore), accept.

After scaffolding completes, install clsx:

```bash
npm install clsx
```

Then apply these post-scaffold adjustments:

1. **next.config.ts** -- Add `eslint.ignoreDuringBuilds: true` so builds don't fail looking for ESLint (Biome handles linting):

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  eslint: {
    ignoreDuringBuilds: true,
  },
};

export default nextConfig;
```

2. **Verify globals.css** uses Tailwind v4 syntax (`@import "tailwindcss"`). If it has the old v3 `@tailwind base/components/utilities` directives, replace with:

```css
@import "tailwindcss";
```

3. **Verify biome.json** exists and has linter enabled. Review the generated config -- no changes needed unless linter is disabled. Do NOT create a tailwind.config.js file (Tailwind v4 uses CSS-first config).

4. **Review package.json scripts** -- ensure `"lint"` script uses biome (e.g., `"biome check"` or `"biome lint"`). If it still references `next lint`, update it to `"biome check src/"`. Also add a format script if missing: `"format": "biome format --write src/"`.
  </action>
