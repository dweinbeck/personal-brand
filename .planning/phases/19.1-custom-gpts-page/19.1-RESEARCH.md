# Phase 19.1: Custom GPTs Page - Research

**Researched:** 2026-02-08
**Domain:** Next.js static page, data-driven card grid, navigation integration
**Confidence:** HIGH

## Summary

This phase adds a new public page at `/custom-gpts` that displays Custom GPT entries as a responsive card grid. The codebase already has every pattern needed -- there is nothing new to build from scratch. The work is purely compositional: create a data file, a page component, and a nav entry, all following existing conventions.

The primary patterns to follow are:
1. **Data file pattern:** `src/data/accomplishments.json` + `src/lib/accomplishments.ts` (JSON array + typed loader)
2. **Page layout pattern:** `src/app/building-blocks/page.tsx` and `src/app/about/page.tsx` (server component, metadata export, responsive grid)
3. **Card pattern:** The existing `Card` component at `src/components/ui/Card.tsx` already handles external links with `target="_blank" rel="noopener noreferrer"` when `href` starts with `http`
4. **Navigation pattern:** `src/components/layout/NavLinks.tsx` uses a `baseLinks` array with `{ name, href }` objects

**Primary recommendation:** Use the existing `Card` component with `variant="clickable"` and external `href` props. Source data from `src/data/custom-gpts.json` with a typed loader in `src/lib/custom-gpts.ts`. Add a nav entry to `NavLinks.tsx`. No new dependencies needed.

**When to use:** Static or semi-static data that doesn't come from an API

  slug: string;

**Source:** Existing pattern in `src/lib/accomplishments.ts` + `src/data/accomplishments.json`

### Pattern 3: Card Component with External Links
**What:** The `Card` component already detects external URLs and renders `<a target="_blank" rel="noopener noreferrer">`
**When to use:** Any card that links externally
**Key detail:** When `href` starts with `http`, Card automatically adds `target="_blank" rel="noopener noreferrer"` (line 39-49 of Card.tsx). No manual external link handling needed.
**Source:** `src/components/ui/Card.tsx` lines 38-49

### Pattern 4: Navigation Entry
**What:** Add entry to `baseLinks` array in NavLinks.tsx
**When to use:** When adding a new public page to site navigation
**Key detail:** The `isActive` function uses `pathname.startsWith(href)` for non-root routes, so `/custom-gpts` will correctly highlight when the user is on that page.
**Source:** `src/components/layout/NavLinks.tsx` lines 10-18, 35-38

### Anti-Patterns to Avoid
- **Don't create a new card component:** The existing `Card` component with `variant="clickable"` provides all needed styling (rounded-2xl, border, shadow, hover effects, translate-y). Render card content as children.
- **Don't hardcode GPT data in the page component:** The brief explicitly requires a separate data file. Follow the `accomplishments.json` pattern.
- **Don't use `"use client"` on the page:** This page has no interactivity. Keep it as a server component (the default in App Router).
- **Don't add `dot-pattern` background class:** The Projects page uses it, but Building Blocks and About pages do not. The standard pattern is the plain background.

## Common Pitfalls

### Pitfall 1: Forgetting External Link Attributes
**What goes wrong:** External links open in same tab or lack security attributes
**Why it happens:** Manually creating `<a>` tags instead of using `Card` component
**How to avoid:** Use `Card` component with `href` prop for external URLs. It automatically adds `target="_blank" rel="noopener noreferrer"`.
**Warning signs:** Links in the card grid that navigate away from the site in the same tab

### Pitfall 2: Navigation Ordering Mismatch
**What goes wrong:** "Custom GPTs" appears in an awkward position in the nav bar
**Why it happens:** Adding it at the end of `baseLinks` without considering logical grouping
**How to avoid:** Place "Custom GPTs" adjacent to "Projects" and "Building Blocks" in the nav since they are content showcase pages. The current order is: Home, About, Projects, Building Blocks, Assistant, Contact. A logical position is after "Building Blocks" (index 4).
**Warning signs:** Nav feels disorganized; related pages aren't grouped together

### Pitfall 3: Missing Placeholder URL Markers
**What goes wrong:** Placeholder URLs look like real GPT URLs, making it hard to identify which need updating
**Why it happens:** Using generic ChatGPT URLs without clear markers
**How to avoid:** Use obviously placeholder URLs like `https://chat.openai.com/g/g-PLACEHOLDER-slug-name` and add a comment in the JSON file header or use a clear naming convention
**Warning signs:** URLs that look plausible but don't work

### Pitfall 4: Inconsistent Page Layout Spacing
**What goes wrong:** Page looks different from Building Blocks / About pages
**Why it happens:** Using different max-width, padding, or grid classes
**How to avoid:** Copy exact container classes from Building Blocks page: `mx-auto max-w-5xl px-4 sm:px-6 lg:px-8 py-16`. Use exact grid classes: `mt-10 grid grid-cols-1 md:grid-cols-2 gap-6`.
**Warning signs:** Page feels "off" compared to sibling pages

### Pitfall 5: Card Content Styling Inconsistency
**What goes wrong:** Card internals don't match the visual style of TutorialCard or ProjectCard
**Why it happens:** Using different font sizes, spacing, or color classes for title/description/tags
**How to avoid:** Match the proven card content pattern:
- Title: `text-lg font-bold text-text-primary group-hover:text-gold transition-colors duration-200`
- Description: `mt-3 text-sm text-text-secondary leading-relaxed line-clamp-3`
- Tags: `px-2.5 py-0.5 font-mono text-xs text-text-tertiary bg-[rgba(27,42,74,0.04)] rounded-full`
- CTA text: `mt-5 inline-block text-sm font-medium text-gold group-hover:text-gold-hover transition-colors duration-200`
**Warning signs:** Font sizes or colors look different from other card pages

### Page Component
```typescript
// src/app/custom-gpts/page.tsx
import type { Metadata } from "next";
import { Card } from "@/components/ui/Card";
import { getCustomGpts } from "@/lib/custom-gpts";

export const metadata: Metadata = {
  title: "Custom GPTs",
  description:
    "Custom GPTs built by Dan Weinbeck for AI-powered productivity and development.",
};

export default function CustomGptsPage() {
  const gpts = getCustomGpts();

  return (
    <div className="mx-auto max-w-5xl px-4 sm:px-6 lg:px-8 py-16">
      <h1 className="text-3xl font-bold text-text-primary">Custom GPTs</h1>
      <p className="mt-2 text-text-secondary">
        Custom-built GPTs for AI-powered productivity and development.
      </p>
      <div className="mt-10 grid grid-cols-1 md:grid-cols-2 gap-6">
        {gpts.map((gpt) => (
          <Card key={gpt.slug} variant="clickable" href={gpt.url}>
            <h3 className="text-lg font-bold text-text-primary group-hover:text-gold transition-colors duration-200">
              {gpt.name}
            </h3>
            <p className="mt-3 text-sm text-text-secondary leading-relaxed line-clamp-3">
              {gpt.description}
            </p>
            {gpt.tags.length > 0 && (
              <div className="mt-5 flex flex-wrap gap-2">
                {gpt.tags.map((tag) => (
                  <span
                    key={tag}
                    className="px-2.5 py-0.5 font-mono text-xs text-text-tertiary bg-[rgba(27,42,74,0.04)] rounded-full"
                  >
                    {tag}
                  </span>
                ))}
              </div>
            )}
            <span className="mt-5 inline-block text-sm font-medium text-gold group-hover:text-gold-hover transition-colors duration-200">
              Try it &rarr;
            </span>
          </Card>
        ))}
      </div>
    </div>
  );
}
```
**Source:** Composite of `src/app/building-blocks/page.tsx`, `src/components/building-blocks/TutorialCard.tsx`, and `src/components/ui/Card.tsx`
