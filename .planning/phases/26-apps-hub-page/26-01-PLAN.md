```ts
export interface AppListing {
  slug: string;          // URL-friendly identifier
  title: string;         // Display name
  tag: string;           // Topic badge label (e.g., "Branding", "Finance")
  subtitle: string;      // One-line pitch
  description: string;   // 1-2 sentence explanation
  href: string;          // Internal route (e.g., "/apps/brand-scraper")
  launchedAt: string;    // ISO date string or null for unreleased
  updatedAt: string;     // ISO date string or null
  techStack: string[];   // Tech tags shown on card
  available: boolean;    // true = live, false = coming soon
}

Structure (top to bottom inside the card):
1. **Topic badge** (top-left, like TutorialCard's primary tag): colored pill using the `tag` field. Use a simple color map:
   - "Branding" → `bg-gold-light text-gold-hover border-gold`
   - "Finance" → `bg-primary/10 text-primary border-primary/40`

function formatDate(iso: string): string {

    month: "short",

    year: "numeric",
  });
}
```

**1. Create `src/app/apps/page.tsx`** — server component, follow the `custom-gpts/page.tsx` pattern.

```tsx
import type { Metadata } from "next";
import { getApps } from "@/data/apps";
import { AppCard } from "@/components/apps/AppCard";

export const metadata: Metadata = {
  title: "Apps",
  description:
    "Discover and access Dan's web-based tools — from brand identity extraction to budget management.",
};

export default function AppsPage() {
  const apps = getApps();

  return (
    <div className="mx-auto max-w-5xl px-4 sm:px-6 lg:px-8 py-16">
      <h1 className="text-3xl font-bold text-text-primary">Apps</h1>
      <p className="mt-2 text-text-secondary">
        Web-based tools built for real-world use. Some are live, others are on the way.
      </p>
      <div className="mt-10 grid grid-cols-1 md:grid-cols-2 gap-6">
        {apps.map((app) => (
          <AppCard key={app.slug} app={app} />
        ))}
      </div>
    </div>
  );
}
```

This is a server component (no "use client"). No layout.tsx for /apps — it uses the root layout.
